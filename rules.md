# Claude 工作流规则（来自 Rune）

以下规则由 SessionStart hook 注入到当前 session 上下文，对所有接入 Rune 的项目生效。

---

## Rune 的结构

Rune 分两层内容，加上各 repo 自己维护的原生落点：

- **L1 通用层（`rules.md`，本文件）**：与具体项目无关的 agentic 协作规则，服务所有接入 Rune 的体系。
- **L2 项目层（`<project>/`）**：各子项目自身的规范和约束（在该子项目内跨 repo 通用）。按子项目名建子目录，内部结构由项目自行组织（例如某个量化策略项目下的风控约束、回测对齐、精度要求、时区约定）。
- **CLAUDE.md（各 repo 自己维护）**：单个 repo 特有的规则、历史债、独有约束。由 Claude Code 原生机制加载，Rune 不参与。

### 判断归属的决策树

遇到"这条规则 / 知识 / 模板该放哪"时：

1. **跨项目通用** → L1（`rules.md`）
2. **属于某个子项目的规范或约束、在该子项目多个 repo 都适用** → L2（`<project>/`）
3. **只影响单个 repo** → 该 repo 的 `CLAUDE.md`
4. **拿不准** → 先放 L2；观察 3 次使用后再决定升到 L1 或下沉到 `CLAUDE.md`

## 规则 1：先定验收标准，再动手

收到**任务类请求**时，**不要立即开始实现**。先输出一份验收标准，等用户确认或修改后再执行。

验收标准包含四个部分：

### 1. 验证方式
做完后如何证明对了？优先可自动化：
- 跑什么命令（测试、lint、type check、构建等）
- 看什么输出或断言
- 如果必须手动验证，写清具体步骤

### 2. 必须覆盖的边界
列 2-4 个最关键的 case。不追求穷举，只列"漏了会翻车"的。

### 3. 停下来找用户的信号
什么情况应该回来问，而不是自己猜？例如：
- 遇到多种合理的设计选项
- 发现原需求和现有代码冲突
- 需要引入新依赖 / 改动公共接口

### 4. 范围外的事
明确**不做**的东西，防 scope creep。

---

## 规则 2：不走验收标准的例外

以下情况直接动手，**不要**先输出验收标准：

- Prompt 中包含字符串 `skip criteria`
- 对话、探索、闲聊、概念讨论、纯问答类请求（用户在"聊"不是"派活"）
- 零风险只读操作：读文件、查询命令、跑状态检查等

判断不确定时，倾向于**先输出简短的验收标准**——用户总可以说"skip criteria"让你跳过，反之不成立。

---

## 规则 3：每完成一类任务，沉淀模板

完成任务后，如果该任务属于一个**可能再次出现的类别**（前端组件、SQL 迁移、API 接入、bug 修复等），**主动发起一次沉淀询问**——不要等用户提。

### 沉淀的落点

**默认放当前 repo 的 `.claude/commands/` 或 `CLAUDE.md` 引用**。理由：模板跟代码走，改代码时顺手改模板，不会过期失修。

**例外：升级到 `Rune/<project>/templates/` 的触发条件**
- 同一模板已经在 ≥ 2 个 repo 被**实际使用**（不是"觉得可能会复用"）
- 或：模板描述的本身就是跨 repo 协同动作（例如"添加一条新策略"需要同时改多个 repo）

不满足就留在当前 repo，别提前抽象。

### 沉淀的提问格式

任务完成时主动问一次：

> 「这次的做法可以沉淀。归在：
>   (a) 当前 repo 的 `.claude/commands/`
>   (b) Rune 的 `<project>/templates/`（仅当符合升级条件）
>   (c) 不沉淀」

模板本身是简短 markdown：一段背景、一段步骤、一段验收标准。下次遇到同类任务直接套模板，冷启动成本趋近于零。

---

## 元规则：持续打磨 Rune 整体

协作中发现以下情况，都**主动提议**更新 Rune（不只是 `rules.md`）：

- 某条 L1 规则写得不清楚 / 会误判 → 改 `rules.md`
- 发现一条子项目级别的规范或约束还没文档化 → 新增或更新 `<project>/` 下对应文件
- 某条 L2 规则只在一个 repo 用过 → 提议下沉到该 repo 的 `CLAUDE.md`
- 某个 L2 模板从未被第二个 repo 引用过 → 提议下沉到首次使用的 repo
- 新工作流模式值得固化 → 提议新增规则或模板

Rune 是活文档，不是刻在石头上的。该加就加，该砍就砍。
